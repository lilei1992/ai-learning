01 / What It Is

Claude Code 是什么？

一个运行在开发环境里的 Agent：能读项目、改文件、跑命令、看结果，然后继续调整。

不是聊天框聊天机器人给建议；Claude Code 可以形成行动闭环。
不是脚本脚本路径固定；Agent 会根据反馈重新判断下一步。
不是 IDE 插件它更像一个命令行执行系统，UI 只是入口。
不是魔法核心是上下文、工具、权限和可恢复执行。
用“实习工程师 + 工具箱 + 审批制度”类比，先让小白理解 Agent 与 Chatbot 的区别。

03.1 / Code Shape

高级之处：流式循环，而不是一次性回答

query.ts / conceptual sketch

        export async function* query(params: QueryParams) {
          let state = createInitialState(params)

          while (!state.done) {
            const context = assembleContext(state)
            const stream = callModel(context)

            for await (const event of stream) {
              if (event.type === "tool_use") {
                const decision = await authorize(event.tool)
                const result = await executeTool(event.tool, decision)
                state = appendResult(state, result)
              }
              yield event
            }

            state = checkStopCondition(state)
            state = maybeCompactContext(state)
          }
        }
强调 Async Generator 的价值：模型输出、工具结果、UI 事件、错误恢复可以统一进入一个流。
04 / Context Engineering

上下文工程：Agent 的短期工作台

01 Input用户目标、系统规则、项目规范、历史会话assemble
02 Retrieval按任务读取文件、搜索代码、提取局部证据select
03 Budget控制 token、截断工具结果、避免上下文爆炸budget
04 Compact把长会话压成摘要，同时保留决策依据compress
05 Resume从 transcript/checkpoint 恢复任务现场recover
讲“上下文不是越多越好”，而是把正确的信息放到正确的位置。
05 / Tool Calling

工具调用：让模型的意图变成动作

Read只读，可并发
Grep搜索，可并发
Glob文件匹配，可并发
WebFetch外部读取
Edit写入，独占
Write覆盖，高风险
Bash命令，需审批
Agent委托，隔离上下文
Schema工具声明输入结构，让模型知道怎么调用。
Policy按工具、路径、命令和上下文判断风险。
Approval不确定或高风险动作交给人确认。
Execute只读并发，写操作串行，危险命令隔离。
Observe结果回写 transcript，再进入下一轮判断。
强调并发安全：不是所有工具都一样，读写要分开，危险动作要审批。

07 / Memory System

记忆系统：让 Agent 不从零开始

Project Memory项目规则、架构约定、运行命令、编码规范。
Session Memory当前任务的计划、进度、工具结果和关键决策。
Long-term Memory用户偏好、历史方案、事故复盘、团队知识库。
Files稳定规则写入 AGENTS.md / CLAUDE.md。
DB结构化、权限化、可审计的业务记忆。
Vector大量非结构化资料检索。
Transcript任务可回放、可诊断、可恢复。
Policy敏感信息脱敏，不把 secret 交给模型。
记忆不建议只选一种存储。不同记忆类型应该进入不同系统。

09 / Best Practices

最佳实践：把 Agent 当成生产系统

最小权限MCP Server 不拿全局超级权限，工具按资源、动作和用户身份隔离。
读写分离查询类、变更类、危险操作类工具分开注册，降低误执行风险。
人工审批退款、库存、支付、订单状态等真实业务后果必须二次确认。
可观测性记录任务 ID、traceId、工具调用链、参数摘要、异常和审批记录。
可恢复长任务保存 plan、progress、checkpoint，失败后能从关键步骤重放。
敏感脱敏禁止把 token、手机号、身份证、银行卡等原样返回给模型。
这里可以结合公司内部接入 Claude Code / AI Agent 的治理问题。
10 / Future

未来趋势：从工具助手到工程操作系统

01主动式 Agent从等待输入，转向主动感知、规划和提出变更。
02多 Agent 协调像 MapReduce 一样，把搜索、修改、验证并行化。
03长期记忆项目知识、组织规范和历史决策成为一等公民。
04治理内建权限、审计、审批、风控成为 Agent 平台标配。
结尾强调：未来不是让 AI 随便接管，而是构建可信、可控、可审计的 Agent 基础设施。

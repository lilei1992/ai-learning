核心目标：

- 掌握如何把大任务拆给子 Agent。

关键概念：

- `Task Decomposition（任务拆解）`：把复杂目标拆成可独立执行的子任务。
- `Task Boundary（任务边界）`：子任务的输入、输出和范围。
- `Dependency Graph（依赖图）`：子任务之间的依赖关系。
- `Parallelizable Task（可并行任务）`：不依赖彼此结果的任务。

需要理解：

- 不是所有任务都能并行。
- 子任务描述要具体，否则子 Agent 会跑偏。
- 子任务输出格式要预先约定。

今日输出：

- 把“重构一个订单服务”拆成 5 个子任务，并标注哪些可并行。

自测问题：

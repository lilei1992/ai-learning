# Week 05：State、Permission 与 Guardrails QA

> 复习目标：理解 Agent 的安全和可靠性来自 Runtime 约束，而不是模型自觉。

## Day 29：AI Action Authorization

### Q1：模型提出动作后，谁决定能不能执行？

Runtime 的权限系统决定。模型只有建议权，没有执行授权。

### Q2：动作授权要看哪些因素？

工具类型、输入参数、资源范围、风险等级、用户权限、是否需要人工审批。

### Q3：为什么动作要结构化？

结构化后才能做策略判断、审批、审计和重放。

## Day 30：Permission Pipeline

### Q1：权限管线包含哪些步骤？

解析动作、风险分类、策略匹配、权限判断、人工确认、执行或拒绝、记录审计。

### Q2：低风险和高风险动作区别是什么？

低风险多为只读；高风险会修改文件、数据、外部系统或发送消息。

### Q3：拒绝动作后要做什么？

生成 Observation，说明拒绝原因和可选替代方案，让模型调整策略。

## Day 31：Sandbox

### Q1：Sandbox 解决什么问题？

限制工具影响范围，防止越权访问、误删、误执行和敏感数据泄露。

### Q2：文件工具的沙箱边界是什么？

限制 workspace 根目录、禁止路径穿越、写操作需要审批、敏感文件默认拒绝。

### Q3：命令工具的沙箱边界是什么？

限制 cwd、命令白名单、环境变量脱敏、网络限制、超时和输出大小。

## Day 32：Hooks

### Q1：Hook 在 Runtime 中有什么用？

在动作执行前后插入校验、审计、脱敏、限流、通知和策略判断。

### Q2：常见 Hook 有哪些？

`beforeToolCall`、`afterToolCall`、`beforeWrite`、`afterFailure`、`beforeFinalAnswer`。

### Q3：Hook 和 Permission 有什么区别？

Permission 决定能不能做；Hook 是在流程关键点执行附加逻辑。

## Day 33：Audit Log

### Q1：审计日志要记录什么？

任务、用户、动作、工具、输入摘要、权限决策、结果、错误、时间、耗时和风险等级。

### Q2：审计日志不能记录什么？

token、密码、cookie、私钥、授权头、个人隐私和敏感生产数据。

### Q3：审计日志和 Transcript 有什么关系？

Transcript 是执行事实账本，审计日志是面向安全和合规的关键事件记录。

## Day 34：Security Failure Modes

### Q1：Agent 常见安全失败有哪些？

Prompt Injection、越权工具调用、误删文件、误发消息、误执行命令、泄露敏感信息。

### Q2：如何防 Prompt Injection？

把工具返回当作不可信内容，隔离来源，保留系统规则优先级，禁止外部内容改变权限策略。

### Q3：高风险操作为什么要人工确认？

因为后果可能不可恢复，且模型可能误判意图、路径或业务语义。

## Day 35：Week 5 复盘

### Q1：安全版 Runtime 的核心模块有哪些？

State Store、Permission Service、Risk Classifier、Approval Flow、Sandbox、Audit Log、Recovery。

### Q2：权限、幂等、验证之间是什么关系？

权限防越权，幂等防重复副作用，验证证明状态真的符合目标。

### Q3：本周最重要的一句话是什么？

模型输出不是授权，所有动作必须经过 Runtime 的安全管线。

## Week 5 认知升级问题

### Q1：第五周最重要的思想是什么？

安全不是靠模型自觉，而是靠 Runtime 把动作放进权限、风险、沙箱、审批和审计的管线里。

### Q2：为什么说模型输出不是授权？

模型输出只是候选动作。授权必须基于用户身份、资源范围、动作风险、当前状态和组织策略，由 Runtime 判断。

### Q3：风险分类为什么要看副作用？

因为真正的风险来自外部状态变化。读文件和删文件、查日志和改数据库、生成邮件草稿和直接发送邮件，风险等级完全不同。

### Q4：为什么拒绝动作也要生成 Observation？

模型需要知道动作为什么被拒绝，才能选择安全替代方案。否则它可能反复提出同一个危险动作。

### Q5：Sandbox 能解决所有安全吗？

不能。Sandbox 只能限制影响范围，还需要权限策略、审批、审计、脱敏、超时、输出限制和人工确认。

### Q6：审计日志为什么不能等同于普通日志？

普通日志偏排障，审计日志偏责任链和合规。它要记录动作、权限决策、审批人、资源范围、结果和时间，但要避免泄露敏感信息。

### Q7：高风险动作审批时应该给用户看什么？

应该展示动作目的、影响资源、参数摘要、风险等级、预期副作用、可回滚性和拒绝后的替代方案。

### Q8：第五周学懂的标志是什么？

你能给任意工具调用判断风险等级，并设计执行前校验、执行中沙箱、执行后审计和失败恢复策略。

## 4. 最小心智模型

可以先记住这句话：

> Agent 每一步为什么需要审计日志？ 这个问题的答案，最终都要落到“如何让 Agent 更可控、更准确、更可验证”。

从 Runtime 视角看，今天主题和下面链路有关：

```text
User Goal
-> Context / State
-> Model Decision
-> Runtime Control
-> Tool / Memory / Permission / Trace
-> Observation
-> Next Step
```

不要只问“模型会不会”，要问：

- Runtime 给模型看了什么？
- 模型输出如何被解析和校验？
- 工具或状态是否真的发生变化？
- 失败时有没有记录和恢复？
- 最终结论有没有证据？

## 5. 核心概念拆解

### 5.1 Audit Log（审计日志）

记录关键动作和权限决策。

它解决的问题：让 `Audit Log（审计日志）` 从抽象术语变成 Runtime 可以判断、记录和执行的工程边界。

工程落点：通常需要在状态字段、策略规则、接口 Schema、执行流程或审计记录中明确表达 `Audit Log（审计日志）`。

忽略后果：如果 `Audit Log（审计日志）` 只停留在 prompt 里，Agent 容易出现误执行、假成功、不可恢复或不可追踪的问题。

### 5.2 Trace（链路）

记录执行全过程。

它解决的问题：让 `Trace（链路）` 从抽象术语变成 Runtime 可以判断、记录和执行的工程边界。

工程落点：通常需要在状态字段、策略规则、接口 Schema、执行流程或审计记录中明确表达 `Trace（链路）`。

忽略后果：如果 `Trace（链路）` 只停留在 prompt 里，Agent 容易出现误执行、假成功、不可恢复或不可追踪的问题。

### 5.3 Sensitive Data（敏感数据）

日志中不能出现 token、密码等。

它解决的问题：让 `Sensitive Data（敏感数据）` 从抽象术语变成 Runtime 可以判断、记录和执行的工程边界。

工程落点：通常需要在状态字段、策略规则、接口 Schema、执行流程或审计记录中明确表达 `Sensitive Data（敏感数据）`。

忽略后果：如果 `Sensitive Data（敏感数据）` 只停留在 prompt 里，Agent 容易出现误执行、假成功、不可恢复或不可追踪的问题。

### 5.4 Evidence（证据）

审计必须能证明发生过什么。

它解决的问题：让 `Evidence（证据）` 从抽象术语变成 Runtime 可以判断、记录和执行的工程边界。

工程落点：通常需要在状态字段、策略规则、接口 Schema、执行流程或审计记录中明确表达 `Evidence（证据）`。

忽略后果：如果 `Evidence（证据）` 只停留在 prompt 里，Agent 容易出现误执行、假成功、不可恢复或不可追踪的问题。

## 6. 工程含义

今天主题的工程含义可以分成 5 层：

1. **边界**：明确模型、Runtime、工具、状态、用户各自负责什么。
2. **结构**：用接口、Schema、状态机、表结构或日志结构把能力固定下来。
3. **安全**：对高风险动作设置权限、审批、沙箱或只读限制。
4. **可恢复**：失败后能重试、降级、停止或交给用户处理。
5. **可验证**：最终结论必须能从工具结果、日志、状态或测试中找到证据。

## 7. Java / 后端类比

像订单、支付系统必须有操作日志，否则无法追责和排查。

你可以用下面的问题检查自己是否真的理解：

- 如果把它做成一个 Spring Bean，它的输入输出是什么？
- 它应该依赖哪些组件，不应该依赖哪些组件？
- 它的失败异常应该抛出、重试、降级还是记录？
- 它会不会影响数据库、Redis、MQ、ES 或外部系统状态？

## 8. 设计清单

学习今天主题时，至少检查这些设计点：

- 是否有清晰的输入和输出。
- 是否有结构化数据，而不是只靠自然语言。
- 是否能被记录到 Transcript / Trace。
- 是否能区分成功、失败、拒绝、超时和部分成功。
- 是否需要权限控制。
- 是否需要幂等或重试。
- 是否会污染上下文或 Memory。
- 是否能被测试和回放。

### Q3：今天最容易出错的工程点是什么？

## 11. 常见坑

- 只会解释概念，但说不出它在 Runtime 里的位置。
- 只相信模型输出，没有结构化校验。
- 没有考虑失败、超时、权限和审计。
- 把所有信息都塞进上下文，导致模型被噪声干扰。
- 没有最终验证，却在回答里声称任务完成。

## 12. 今日总结

今天真正要记住的是：

> Agent 工程化不是让模型“更自由”，而是让模型的推理能力被 Runtime 安全、结构化、可追踪地使用。

## 13. 补充深度学习内容

### 13.1 Audit Log 是责任链

审计日志要回答：

- 谁发起了任务？
- 模型提出了什么动作？
- Runtime 为什么允许或拒绝？
- 谁审批了高风险动作？
- 工具实际执行了什么？
- 结果如何？

### 13.2 字段设计

```text
taskId, userId, actionId, toolName, riskLevel,
inputSummary, resourceScope, permissionDecision,
approver, resultStatus, errorCode, rawRef, createdAt
```

### 13.3 注意脱敏

审计要完整，但不能记录 token、cookie、密码、私钥、Authorization Header。

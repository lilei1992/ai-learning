# Week 06：Agent 架构模式与 MCP 扩展 QA

> 复习目标：为不同任务选择合适 Agent 模式，并理解 MCP 如何标准化工具扩展。

## Day 36：MCP 基础

### Q1：MCP 解决什么问题？

MCP 标准化模型应用与外部工具、资源之间的连接方式，让 Agent 能发现和调用外部能力。

### Q2：MCP 的核心角色有哪些？

Host、Client、Server、Tool、Resource。

### Q3：MCP 的工程价值是什么？

工具扩展标准化、进程隔离、能力发现、跨应用复用。

## Day 37：JSON-RPC

### Q1：JSON-RPC 是什么？

一种用 JSON 表达远程方法调用的协议，包含 method、params、id、result、error。

### Q2：为什么 MCP 使用类似 JSON-RPC 的通信模式？

结构简单、语言无关、适合进程间工具调用。

### Q3：错误响应为什么重要？

Runtime 需要根据错误类型决定重试、降级、禁用插件或询问用户。

## Day 38：Process Isolation

### Q1：为什么工具要进程隔离？

防止插件崩溃拖垮主进程，也限制权限范围和资源影响。

### Q2：隔离能降低哪些风险？

越权访问、依赖冲突、内存泄漏、崩溃传播、敏感数据泄露。

### Q3：隔离不是万能的原因是什么？

仍需要权限控制、输入校验、输出脱敏和审计。

## Day 39：Resource vs Tool

### Q1：Resource 和 Tool 的区别是什么？

Resource 主要提供可读取上下文；Tool 代表可执行动作，可能产生副作用。

### Q2：为什么要区分二者？

读上下文和执行动作风险不同，权限策略也不同。

### Q3：例子是什么？

项目 README 是 Resource；执行测试、写文件、查询数据库是 Tool。

## Day 40：Plugin Architecture

### Q1：插件架构需要哪些信息？

名称、版本、能力、工具列表、资源列表、权限、启动方式、超时和错误策略。

### Q2：插件为什么要版本化？

工具输入输出变化会影响 Agent 行为，需要兼容和回滚。

### Q3：插件权限为什么要最小化？

减少越权和误操作影响范围。

## Day 41：MCP Error Handling

### Q1：MCP Server 失败时 Runtime 怎么办？

记录错误、生成 Observation、按策略重试、降级、禁用插件或请求用户处理。

### Q2：插件超时如何处理？

中断调用，记录超时，必要时重试只读幂等操作。

### Q3：插件返回异常数据怎么办？

做 schema 校验，拒绝污染上下文，并记录为失败 Observation。

## Day 42：Week 6 复盘

### Q1：ReAct 和 Plan-and-Execute 如何选择？

探索任务用 ReAct；长任务且目标可分解时用 Plan-and-Execute。

### Q2：什么时候用 Workflow + Agent？

当任务既需要模型理解，又涉及高风险确定性业务流程时。

### Q3：本周最重要的一句话是什么？

Agent 架构选型要服务任务风险和复杂度，不要为了“智能”过度设计。

## Week 6 认知升级问题

### Q1：第六周最重要的思想是什么？

扩展能力必须协议化、隔离化、可治理。不能因为插件“有用”，就让它无限制接入 Runtime。

### Q2：MCP 解决的核心问题是什么？

它提供一种标准方式，让 Runtime 发现、描述和调用外部工具与资源，降低每个工具单独适配的成本。

### Q3：为什么 Tool 和 Resource 要区分？

Tool 可能产生副作用，需要权限和审计；Resource 主要提供上下文，需要关注可信度、缓存和引用来源。

### Q4：为什么插件应该进程隔离？

插件可能崩溃、阻塞、泄露信息或包含不可信代码。进程隔离能保护 Runtime 主进程，并方便超时、重启和权限限制。

### Q5：JSON-RPC 在 MCP 中有什么价值？

它提供请求、响应、错误、通知的统一消息格式，让 Runtime 和 MCP server 可以跨进程通信。

### Q6：插件系统最容易出现什么治理问题？

工具重名、权限过大、版本不兼容、Schema 变化、错误吞掉、输出污染上下文、供应链不可信。

### Q7：MCP server 返回错误时 Runtime 应该怎么做？

结构化记录错误，判断是否可重试，必要时降级或禁用插件，并把可恢复信息转成 Observation。

### Q8：第六周学懂的标志是什么？

你能设计一个插件接入流程：声明能力、校验权限、启动进程、调用协议、处理错误、记录审计、支持禁用和升级。

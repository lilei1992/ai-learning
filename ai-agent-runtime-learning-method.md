# AI Agent Runtime 学习法执行手册

> 创建日期：2026-06-02
> 适用计划：[`ai-agent-runtime/README.md`](ai-agent-runtime/README.md)
> 目标：把 56 天 Agent 学习计划从“看材料”升级为“能复述、能设计、能落地”。

## 1. 核心原则

每天只追求一个结果：

> 学完当天主题后，能不用资料解释核心概念，并能把它映射到 Java / 后端工程经验。

判断是否真正学会，不看阅读时长，看下面 4 个结果：

- 能用自己的话讲清楚。
- 能回答自测问题。
- 能写出一个 Java / 后端类比。
- 能留下一个可复用产出，例如图、伪代码、接口草图、检查清单。

## 2. 每日学习闭环

每天按这个顺序执行：

```text
预习问题
-> 阅读核心内容
-> 主动回忆
-> 费曼输出
-> Java / 后端类比
-> 小练习
-> 间隔复习登记
```

### 2.1 预习问题：先带问题进入

阅读前先看当天标题和“今日核心目标”，写下 1 个问题。

示例：

```markdown
### 预习问题

- Agentic Loop 为什么不能只是一个 while 循环？
```

作用：

- 避免无目标阅读。
- 帮你判断今天的重点。
- 让大脑先建立检索线索。

### 2.2 阅读核心内容：只抓主干

阅读时优先抓 3 类信息：

- 定义：这个概念是什么。
- 边界：它不负责什么。
- 工程问题：它解决什么生产问题。

不建议第一遍就追求完整细节。Agent 相关知识很容易发散，第一遍只建立主干。

### 2.3 主动回忆：不看资料回答

读完后合上资料，回答当天自测问题。

最低要求：

```markdown
### 主动回忆

1. 今天的核心概念是什么？
2. 它解决了什么工程问题？
3. 如果没有这个机制，会出现什么风险？
```

判断标准：

- 能答出来：说明已经形成初步理解。
- 答不完整：回到对应小节补读。
- 只能照抄原文：说明还没转成自己的知识。

### 2.4 费曼输出：用 5 句话讲给后端同事

每天写一段 5 句话以内的解释，假设对方懂 Java / Spring Boot，但不懂 AI Agent。

模板：

```markdown
### 费曼输出

1. 今天这个概念解决的是：...
2. 它在 Agent Runtime 里的位置是：...
3. 如果用 Java / Spring 类比，它像：...
4. 它最容易出问题的地方是：...
5. 我现在还不确定的是：...
```

要求：

- 不堆术语。
- 不复制原文。
- 每句话都能独立表达一个判断。

### 2.5 Java / 后端类比：把新知识挂到旧经验上

每天至少写 1 个类比。

常用类比方向：

| Agent 概念 | Java / 后端类比 |
|------------|------------------|
| Agentic Loop | 长事务任务编排 / 状态机驱动流程 |
| Tool Registry | SPI 注册表 / Bean 容器 |
| Tool Schema | DTO validation / OpenAPI schema |
| Permission Pipeline | Spring Security FilterChain |
| Context Window | 有限内存 / 请求上下文 |
| Context Compaction | GC / 日志压缩 / 快照 |
| Observation | 外部服务调用结果 / trace event |
| MCP Server | 独立 RPC 服务 / 插件进程 |
| Multi-Agent | ForkJoinTask / CompletableFuture worker |
| Evaluation | 自动化测试 + 监控指标 + 审计 |

类比不是为了完全等价，而是为了快速建立工程直觉。写类比时要同时写出“不一样的地方”。

### 2.6 小练习：当天必须有一个可执行动作

根据学习模式选择练习强度。

| 模式 | 小练习 |
|------|--------|
| Busy Mode | 写 3 条要点 + 1 个问题 |
| Standard Mode | 写 5 句话费曼输出 + 1 个 Java 类比 |
| Deep Mode | 画图 / 写伪代码 / 设计接口草图 / 做检查清单 |

示例输出：

```markdown
### 今日小练习

- 画出 `Goal -> Agentic Loop -> Tool -> Observation -> Next Step` 流程。
- 写一个 `Tool` interface 草图。
- 列出 5 个 BashTool 的安全风险。
```

## 3. 间隔重复规则

Agent 学习不能只靠当天理解。每个主题至少复习 3 次。

| 复习点 | 时间 | 做什么 |
|--------|------|--------|
| D+1 | 下一次学习前 | 不看资料，用 2 句话复述昨天主题 |
| D+3 | 3 天后 | 回答昨天/前几天留下的问题 |
| D+7 | 每周 Review Day | 把本周概念串成一张图或一段伪代码 |

每天笔记底部增加：

```markdown
### 间隔复习

- D+1：
- D+3：
- D+7：
```

如果当天没时间，只做 D+1 复述，也比重新阅读更有效。

## 4. 每周复盘方法

每周第 7 天是 Review Day，不学新概念，主要做整合。

复盘顺序：

```text
回看本周 6 个主题
-> 按依赖关系重排概念
-> 画一张总图
-> 写一段伪代码或接口草图
-> 列出 3 个仍模糊的问题
```

每周固定产出：

- 一张结构图：模块之间如何连接。
- 一个工程类比：和 Java / 后端系统的对应关系。
- 一个风险清单：生产落地最容易失败的地方。
- 一个下周问题：下一周要重点验证什么。

建议从 Day 03 开始，每天在“今日笔记”下补充这些小节：

```markdown

- D+7：
```

Day 01 和 Day 02 已经有你的学习痕迹时，不要强行重写。可以只在底部追加缺失小节。

## 6. 忙碌时的最低完成标准

如果当天只有 15 分钟，按这个顺序：

1. 看当天“今日核心目标”。
2. 阅读核心概念。
3. 不看资料写 3 条要点。
4. 写 1 个还没想清楚的问题。

忙碌模式也算完成，但本周 Review Day 要补：

- 费曼输出。
- Java / 后端类比。
- 小练习。

## 7. 不建议的学习方式

避免这些行为：

- 连续阅读很多天，但没有任何输出。
- 收藏资料代替学习。
- 直接看框架源码，但还没理解 Agentic Loop、Tool、Context、Permission 的边界。
- 用模型名、产品名替代底层概念。
- 每天都追求完整，导致中断后不再继续。

更好的方式：

- 小步推进。
- 每天留痕。
- 周复盘整合。
- 先掌握稳定工程抽象，再看具体产品实现。
